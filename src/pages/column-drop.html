Responsive Design and adaptative layout patterns

RWD is an approach to web design aimed at allowing desktop webpages to be viewed in response to the size of the screen or web browser one is viewing with. Thus, using responsive design techniques, webpages layout are enabled to adapt to a variety of screen sizes.

A key part of this design approach is the use of fluid, proportion-based grids. The fluid grid concept calls for page element sizing to be in relative units like percentages, rather than absolute units like pixels or points. For that purpose, space is divided in a row-column fashion schema where several elements can share the same row by spanning one or multiple columns.

Implementation of responsive design has led to a common set of layouts frequently used by designers. From a high-level perspective, we could classify them into 5 categories: mostly fluid, column drop, layout shifter, tiny tweaks and off canvas

Column Drop Layout Pattern credits to Luke Wroblewski

Another popular pattern starts with a multi-column layout and ends up with a single column layout, dropping columns along the way as screen sizes get narrower. Unlike the Mostly Fluid pattern, the overall size of elements in this layout tend to stay consistent.

Column drop behaviour illustration

Adapting to various screen sizes relies on stacking columns. When and how each column is stacked for each resolution breakpoint will depend on the web design itself.

Key Facts

The following key facts apply for this pattern:

Fluid Grid

The Layout of this site is based on Bootstrap responsive grid.

Multi-Column Layout

Main page is divided into 2 column. Article section spans 8 units while Key Facts section spans 4.

Full Width

The whole page extends to the viewport width, growing and shrinking with it.

Progressive stacking

When reducing size, each visual breakpoint will determine a new column to be stacked.
